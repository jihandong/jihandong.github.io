archlinux安装配置YCM

JiHandong's Blog
2019-07-07
YCM, archlinux, vim

介绍

​ 我对配置的观点是：尽可能少。对于文本编辑器，过多的配置会让人对环境产生过多的依赖，对改变环境不太方便，但是自动补全实在是太诱人了，所以将vim几乎是最好、最难安装、最难配置的自动补全插件YCM的安装配置过程记录如下。
​ 环境：archlinux-5.2，无图形界面

解决依赖

​ 编译安装YCM需要足够高版本的vim，python，clang，cmake，由于我们可以pacman滚动更新，所以这并不构成问题。

​ 关键在于编译还需要一些python包，如果缺少这些包，可以在下载YCM后，在其目录下使用python run-tests.py测试时发现；测试时间很长，我们可以提前准备好。

​ 于是准备就绪，可以开始安装了。

download

​ 首先强烈建议安装vundle插件，这是一个管理其它插件的插件，虽然我们不大可能再安装更多插件，但是保持.vim插件目录下的整洁，构建分层绝对不是什么坏事，况且它也不难安装。

​ 然后我们在vim的配置文件.vimrc中做如下设置

​ 之后我们就可以使用vundle安装YCM，下载YCM的时间很长，因为需要从外网上下载几十M的内容，还可能失败，所以这里我认为使用代理比较好，过程不会显示，需要静静等待。

​ 安装完毕之后别忘了l检查一下安装记录，确认安装成功；如果失败的话可以考虑git clone下载YCM，但是vundle还是要保留。

compile

​ 然后就是最困难的环节：编译安装，之前我们不过是下载完毕而已，接下来还需要我们手动编译安装。
​ 首先需要解决python包的依赖，如果完成了前面的步骤，就可以直接python run-tests.py进行检查，如果正确的话会得到百来个ok，以及最终的一个“大OK”。
​ 然后可以用cmake安装，也可以使用自带的py安装脚本，添加不同的参数，表示对不同语言的支持，比如：--clang-completor表示对c-family语言的支持，--go-completer表示对go语言的支持，此外还有java、rust、C#等，不一一枚举。

​ 如果按照上一步骤解决了依赖，这里的编译会经过一段漫长的时间，并最终成功；如果失败就根据信息看看是什么东西忘了安装吧。

configure

​ 我们已经下载，编译，安装完毕，但还是不能用，我们还要在vim中做一些配置。

​ 还没完呐！YCM自动补全的机制需要一个flags数组，这个数组中包含了让YCM生效的重要信息，尤其是自动补全检索的目录，除此之外还有很多需要编写的东西，YCM中的配置文件没有一个能拿来用的，下面我给出一个模板，可以根据实际电脑还有工程项目调整：
