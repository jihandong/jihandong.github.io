    Introduction

    本次实验的目的是为我们的OS编写内存管理代码，分为两部分：

    1. physical memory allocator：维持一个数据结构，记录物理页的分配情况、共享情况，并允许分配；注意一个页的大小是4096bytes。

    2. virtual memory manager：将虚拟地址map到物理地址。
      使用命令创建并切换分支到lab2，以后的实验里可以不带参数b切换分支。

      本实验需要重点关注新提供的几个文件：
      inc/memlayout.h：struct PageInfo保存了物理页的metadata，与物理页有一一对应的关系，可以利用pp_link找到下一个空闲界面；。

      ，尤其是inc/memlayout.h和kern/pmap.h；后者定义了PageInfo structure，可用于追踪空闲的物理页。

    Part1 物理页管理

    实现我们的physical page allocator，它通过一个struct PageInfo链表追踪所有的页。

    Excercise1

    在文件kern/pmap.c中实现下面的函数boot_alloc()、mem_init()、page_init()、page_alloc()、page_free()。check_page_free_list()、check_page_alloc()测试physical page allocator
    阅读xv6手册chap2，我们知道了xv6的虚拟地址究竟如何通过页表机制映射为物理地址。页表以tow-level tree的形式存储在物理内存中，其根为4096-byte page directory，连接1024个页表页（page tabe pages），每个页表都是1024个32bit的array。其中每个虚拟地址的前10位用于查询page directory，如果entry存在，便用接下来的10位查询对应的oage table，最后的12位作为偏移保留，如此便得到了最后的20位物理地址。mmu.h中有虚拟地址的具体定义。此外这是多级页表的一种典型实现。
    每个进程都有自己的two-level 页表，所以我们需要一个allocator在运行时动态分配物理内存。我们的allocator有一个链表来追踪空闲的物理页，但是在刚开机时，这个链表尚未初始化。JOS的解决方法是额外使用一个allocator，直接在内核的data segment end分配物理内存。这就是我们的下面要编写的一系列函数。
    具体到我们需要编写的函数；开机时，JOS需要使用boot_alloc()分配出two-level页表需要的物理空间，page_init()初始化page_free_list之后就可以使用page_alloc()来分配新的物理页。
    boot_alloc()：只用于JOS初始化时的物理内存分配。
    mem_init()：只用于设置JOS的two-level页表。
    page_init()：初始化空闲页链表，自此可以使用page_*()进行内存管理。
    page_alloc()：分配一个物理页，涉及对page_free_list的操作。
    page_free()：牺牲一个物理页，同样涉及对page_free_list的操作。
